编写一份交接文档（Markdown 格式），总结当前对话，让新的 agent 可以继续工作。

## 保存路径

将交接文档保存到当前工作目录下的 `handoff-temp/` 子目录。

- 如果 `handoff-temp/` 不存在，先创建它。
- 文件名格式：`handoff-<YYYYMMDD-HHMMSS>.md`
- 示例：`handoff-temp/handoff-20250120-143052.md`
- 不要覆盖已有文件。如果同一秒内文件名冲突，在文件名末尾追加短随机后缀，例如 `handoff-20250120-143052-a3f9.md`。

写入前检查目标文件是否已存在。若存在，则按上述规则生成不冲突的新文件名（如追加短随机后缀），然后写入新文件。

## 内容要求

交接文档应让新的 agent 能直接继续工作。优先包含：

1. **对话摘要**：当前讨论的核心问题、已做决定、已完成工作。
2. **当前状态**：已修改或已检查的文件、重要路径、命令、验证结果。
3. **未完成工作**：仍需处理、验证、实现或用户确认的事项。
4. **下一步建议**：建议新的 agent 从哪里开始，优先做什么。
5. **建议使用的 skills**：如适用，列出下一会话建议启用的 skills，避免推荐不存在的 skill。

## 引用已有产物

不要重复已经记录在其他产物中的内容（PRD、计划、ADR、issue、commit、diff）。改用路径或 URL 引用它们。

示例：

> 详见用户提供或仓库中已存在的相关文档路径。
> 参考已确认存在的源码文件或配置文件路径。

不要编造路径、URL、issue 编号或提交哈希。

## 参数处理

如果用户传入了参数，将其视为下一会话的关注点，并据此调整交接文档。

例如（仅作参考，根据实际内容调整）：

- 参数是“修复登录超时 bug”：重点记录复现步骤、错误日志、已排除原因、下一步诊断方向。
- 参数是“继续实现用户注册 API”：重点记录需求、已修改文件、数据库/API 设计、未完成端点。

如果没有参数，则基于当前对话自然生成交接内容。

## 输出

保存完成后，输出交接文件的**相对于当前工作目录的路径**，格式如下：

```text
Handoff saved to: handoff-temp/handoff-<YYYYMMDD-HHMMSS>.md
```

如果用户明确要求绝对路径，先确认当前工作目录，再输出绝对路径。不要猜测、转换或编造宿主机路径。
