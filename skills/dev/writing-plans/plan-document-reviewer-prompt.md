# 计划文档审查者 Prompt 模板

派发计划文档审查子 agent 时使用这个模板。

**目的：**验证计划是否完整、是否匹配规格说明、任务拆分是否合理。

**派发时机：**完整计划写好之后。

```text
Task tool (general-purpose):
  description: "审查计划文档"
  prompt: |
    你是计划文档审查者。请验证这份计划是否完整，是否已经可以交给实现者执行。

    **待审查计划：**[PLAN_FILE_PATH]
    **参考规格说明：**[SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 重点检查 |
    |---|---|
    | 完整性 | TODO、占位符、未完成任务、缺失步骤 |
    | 规格对齐 | 计划是否覆盖规格需求，是否存在明显范围蔓延 |
    | 任务拆分 | 任务边界是否清晰，步骤是否可执行 |
    | 可构建性 | 工程师能否照着这份计划执行，而不会卡住 |

    ## 校准标准

    **只标记会在实现阶段造成真实问题的事项。**
    如果实现者会做错东西或卡住，这就是问题。
    轻微措辞、风格偏好和“有则更好”的建议，不算阻塞问题。

    默认批准，除非存在严重缺口：遗漏规格需求、步骤互相矛盾、存在占位内容，或任务含糊到无法执行。

    ## 输出格式

    ## 计划审查

    **状态：**通过 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为什么会影响实现]

    **建议（仅供参考，不阻塞通过）：**
    - [改进建议]
```

**审查者返回：**状态、问题（如有）、建议。
