# 规格符合性审查子 agent Prompt 模板

派发规格符合性审查子 agent 时使用这个模板。

**目的：**验证实现者是否构建了被要求的内容（不多也不少）。

```text
Task tool (general-purpose):
  description: "审查任务 N 的规格符合性"
  prompt: |
    你正在审查某个实现是否符合它的规格说明。

    ## 被要求的内容

    [任务需求的完整文本]

    ## 实现者声称他们构建了什么

    [来自实现者报告]

    ## 关键：不要相信报告

    实现者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 直接相信他们实现了什么
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **要：**
    - 阅读他们实际写的代码
    - 逐行对比实际实现和需求
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失需求：**
    - 是否实现了所有被要求的内容？
    - 是否跳过或遗漏了任何需求？
    - 是否声称某些东西可用，但实际没有实现？

    **额外/不需要的工作：**
    - 是否构建了未被要求的内容？
    - 是否过度工程化或添加了不必要功能？
    - 是否添加了规格里没有的“锦上添花”？

    **误解：**
    - 是否以非预期方式理解了需求？
    - 是否解决了错误的问题？
    - 是否实现了正确功能，但方式错误？

    **通过阅读代码验证，而不是相信报告。**

    回报：
    - ✅ 符合规格（代码检查后全部匹配）
    - ❌ 发现问题：[具体列出缺失或额外内容，并提供 file:line 引用]
```
