# Skills 技能体系总览

> 当前套装：32 个 skill，7 个目录，已全部汉化。来源为 `superpowers-main`（SP）与 `skills-main` 合成版，并合并了部分 `opencode-skills-chinese`（OC）skill。

---

## 一、体系结构

```
dev/        24 skill   编程全流程（开工→设计→计划→实现→调试→验证→审查→交付→维护）
docs/        3 skill   Word / PDF / 文档协作
meta/        4 skill   创建 skill + 会话管理（压缩 / 交接）
automation/  1 skill   浏览器 E2E
creative/    0         预留：艺术 / 图表 / 前端设计 / 主题
systems/     0         预留：OS 排查 / 网络诊断 / 性能调优
ai/          0         预留：prompt 工程 / RAG / 模型评估
```

---

## 二、A. 开工准备 → `dev/`

给仓库和会话建立基础规则。

| Skill | 来源 | 目录 | 用途 |
|-------|:--:|------|------|
| `setup` | SP | `dev/setup` | 配置 issue tracker、triage 标签、领域文档布局。新项目首次接入时运行。 |
| `using-superpowers` | SP | `dev/using-superpowers` | 合集总规则：每次响应前先匹配 skill，用户指令优先。每次会话自动加载。 |
| `opencode` | OC | `dev/opencode` | 把编码任务委派给 OpenCode CLI，支持长时间自动会话。 |

## 三、B. 需求与设计 → `dev/`

防止一上来就写代码。先想清楚再动手。

| Skill | 来源 | 目录 | 用途 | 产出 |
|-------|:--:|------|------|------|
| `grill-me` | SP | `dev/grill-me` | 高强度追问，暴露方案漏洞 | 决策摘要 |

| `to-prd` | SP | `dev/to-prd` | 把讨论整理成 PRD，发布到 issue tracker | 产品需求文档 |

## 十三、L. 元技能与会话管理 → `meta/`

维护 skill 合集本身，以及对话压缩和上下文交接。

| Skill | 来源 | 目录 | 用途 |
|-------|:--:|------|------|
| `write-a-skill` | SP | `meta/write-a-skill` | 按结构规范创建 / 汉化 / 增强 skill，注重中文本地化。**强调快速创建和格式规范**。 |
| `writing-skills` | SP | `meta/writing-skills` | 用 TDD 方法创建 skill，先验证无 skill 时失败再写。**强调严谨验证和方法论**。 |
| `caveman` | SP | `meta/caveman` | L1/L2/L3 三级压缩，删废话，留技术准确性 |
| `handoff` | SP | `meta/handoff` | 对话压缩成交接文档，新会话继续 |

> **两个入口，各有侧重：** `write-a-skill` 用于快速创建/汉化（管格式），`writing-skills` 用于 TDD 严谨开发（管方法）。不合并。

## 十六、新手推荐路径

```text
 1. setup                         初始化仓库
 2. grill-me                      把想法问清楚
 3. brainstorming                 出设计文档
 4. prototype（可选）              跑原型验证
 5. to-prd                        沉淀需求文档
 6. writing-plans                 拆成实现步骤
 7. using-git-worktrees           创建隔离工作区
 8. test-driven-development       先写测试
 9. executing-plans / subagent-driven-development  按计划执行
10. systematic-debugging          出 bug 系统化排查
11. verification-before-completion 完成前跑验证
12. requesting-code-review + receiving-code-review  审查 + 处理反馈
13. finishing-a-development-branch  合并 / PR / 收尾
```

**辅助：**
- `caveman`：AI 啰嗦时用
- `handoff`：对话太长时用
- `zoom-out`：看不懂代码时用
- `improve-codebase-architecture`：定期架构体检
- `docx` / `pdf`：处理文档时用
- `playwright-cli`：做浏览器自动化时用
