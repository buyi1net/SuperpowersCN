# AI 编程助手新手指南

> 适用人群：会写代码但没用过 AI 编程的人。零基础起步，按周推进。

---

## 第一部分：先搞懂几个概念

### 什么是 AI 编程助手？

你之前的写代码方式：自己敲键盘 → 编译报错 → 查 Stack Overflow

AI 编程之后：**说人话描述需求 → AI 写出代码 → 你审核修改 → 完成**

你不是被替代了，你从"写代码的人"变成了**"审核代码、提需求的人"**。

### 核心名词解释（大白话版）

| 名词 | 白话解释 | 需要专门学吗 |
|------|---------|------------|
| **AI 编程助手** | 一个能在终端里跟你对话、帮你写代码的程序。代表：Claude Code、OpenCode | 这是你要装的工具 |
| **Skill（技能）** | 给 AI 的"工作方法说明书"。比如你输入 `/tdd`，AI 就知道"哦，这次要先写测试再写代码" | **需要**——这就是你要学的核心 |
| **CLAUDE.md** | 项目简介文件。告诉 AI "这个项目用 React + TypeScript，测试用 Vitest"。放项目根目录就行 | **需要**——每个项目都要写 |
| **LSP** | 代码显微镜。让 AI 能"看懂"你的代码结构（知道哪个函数在哪里定义、哪些地方引用了它） | 不用学，工具自带 |
| **MCP** | 给 AI 的"万能插头"。让 AI 能查数据库、调 API、操作浏览器 | 暂时不用管，用到了再说 |
| **Hooks** | 触发器。比如"AI 启动时自动检查有没有更新" | 先别碰，容易搞乱 |
| **Context 窗口** | AI 的"短期记忆"。一次对话能记住的内容有限（约 20 万字），满了就会忘记前面的 | 知道就行 |

## 第二部分：你需要装什么

### 第一步：选一个 AI 编程工具

| 工具 | 推荐理由 | 价格 | 适合谁 |
|------|---------|------|-------|
| **OpenCode** | 开源免费，支持多种 AI 模型（Claude/GPT/Gemini/国产模型），Windows/Mac/Linux 全支持 | **免费** | **新手首选** |
| Claude Code | Anthropic 官方出品，体验最流畅，但只支持 Claude 模型 | 订阅制（$20/月起） | 有预算、追求开箱即用 |

**推荐新手装 OpenCode**：免费、灵活、生态最大。

下载：https://opencode.ai/download

### 第二步：装 Skill 集合

```
装好 OpenCode 后，在终端里执行：

npx skills@latest add mattpocock/skills
```

这步装了你前面了解的那套工作流技能。装完后重启 OpenCode。

### 第三步：写 CLAUDE.md

在你的项目目录里新建一个文件叫 `CLAUDE.md`，写：

```markdown
# 项目名称：我的博客

## 技术栈
- 语言：TypeScript
- 框架：React + Next.js
- 样式：Tailwind CSS
- 数据库：SQLite（通过 Prisma）
- 包管理：pnpm
- 测试：Vitest

## 常用命令
- 启动开发：pnpm dev
- 运行测试：pnpm test
- 构建：pnpm build
```

AI 每次对话都会自动读这个文件。**这是最重要的配置，没有之一。**

## 第三部分：四周学习路线

### 第 1 周：学会"让 AI 干活"

目标：能描述需求让 AI 写出可运行的代码。

**Day 1 — 安装和第一个对话**
1. 下载安装 OpenCode
2. 打开终端，输入 `opencode` 启动
3. 输入："帮我用 Python 写一个计算器，支持加减乘除"
4. AI 会写代码，你看一遍，弄懂每一行

**Day 2 — 学会 /grill-me（追问模式）**
1. 输入：`/grill-me 我想做一个待办事项应用`
2. AI 会追着你问一堆问题（用什么语言、要不要数据库等）
3. 你回答完，AI 说"好，我明白了"
4. 这时候 AI 已经清楚你要什么了，让它开始写

**Day 3 — 学会读 AI 写的代码**
1. 让 AI 写一个功能
2. 对你没看懂的代码段说："解释这行代码在干什么"
3. 关键原则：**不理解就不提交**。AI 写的代码你读不懂，以后出 bug 你修不了

**Day 4 — 学会修 bug（/diagnose）**
1. 故意让 AI 写一个有 bug 的代码（或者自己制造一个）
2. 输入：`/diagnose 这个功能跑起来报错了`
3. AI 会按照：复现 → 分析原因 → 修复 → 验证 的流程来
4. 你跟着这个流程看一遍，以后自己也能分析 bug

**Day 5 — 学会控制 AI 话多的问题**
1. 当 AI 回答太长时，输入：`/caveman`
2. AI 会进入极简模式，只给关键信息
3. 不废话了再提需求

**Day 6 — 续接对话（/handoff）**
1. 当你发现 AI 开始"忘记"前面说过的话（对话太长）
2. 输入：`/handoff`——AI 会把当前讨论的要点压缩成一份简短文档
3. 开新对话，说："接续上一个对话的工作"
4. AI 恢复记忆

**Day 7 — 自己写一个完整的小项目**
1. 从头开始：/grill-me 问清楚需求
2. 让 AI 实现
3. 出 bug 就用 /diagnose
4. 完成了！关键是你理解了每一段代码

### 第 2 周：学会"让 AI 写出靠谱的代码"

目标：让 AI 写出质量稳定、可维护的代码。

**Day 8 — TDD（先写测试再写代码）**
1. 输入：`/tdd 给计算器加一个平方根功能`
2. AI 会先写测试（红），再写代码让它通过（绿），最后重构（重构）
3. 为什么要这样？**先写测试能让 AI 想清楚"我要什么"再动手**

**Day 9 — 代码审查**
1. 让 AI 写一个功能后，说："审查我刚写的那段代码"
2. AI 会找出潜在问题
3. 你主动发现问题的习惯比 AI 帮你写代码的能力更重要

**Day 10 — 理解代码全局（/zoom-out）**
1. 选一个文件/模块，输入：`/zoom-out 解释这段代码和整个项目的关系`
2. AI 会告诉你这个文件在整体中的位置

**Day 11 — 学会让 AI 写设计文档**
1. 输入：`/to-prd 把我们现在讨论的需求整理成一份产品文档`
2. AI 生成一份 PRD（产品需求文档）
3. 以后做复杂功能，先出文档再动手

**Day 12 — 拆任务**
1. 输入：`/to-issues 把上面那份 PRD 拆成可执行的任务`
2. AI 把大任务拆成一个个小任务
3. 你选一个开始做

**Day 13-14 — 项目实战**
做一个小项目（如博客、笔记应用），完整走一遍流程：

```
需求沟通（/grill-me）
  → 出 PRD（/to-prd）
  → 拆任务（/to-issues）
  → 逐个实现（TDD /tdd）
  → 出 bug 修 bug（/diagnose）
  → 审查代码
  → 对话太长就 handoff 续接
```

### 第 3 周：了解更高级的工具（只读，不强制）

目标：知道还有这些东西，但不一定要用。

**了解 superpowers**
- 和 mattpocock 的区别：它更"自动"，不用你手动输 `/xxx`
- install: 按各平台插件市场安装
- 它会自动：你提需求 → 自动追问 → 自动出计划 → 自动执行 → 自动审查
- 你只需要做"决策"：点头或摇头
- **新手别急着重装，先用手动挡理解流程**

**了解 oh-my-openagent**
- 最"黑科技"的——多个 AI 同时干活
- 输入 `ultrawork 做一个登录页面`，它自动调用好几个 AI 角色并行工作
- 提供 LSP 精确编辑、MCP 内置工具
- **也先别急着重装**

**了解 MCP**
- 当你碰到"AI 说没有权限查数据库/读文件/调 API"时
- 搜一下"xxx 的 MCP 服务器"（比如 "PostgreSQL MCP server"）
- 在配置文件里加一行就好
- 初学者不是必须配置

### 第 4 周：根据自己的需求选择升级路线

| 如果你感觉…… | 建议升级路线 |
|------------|------------|
| "手动输命令挺麻烦的" | 换 **superpowers**（自动触发） |
| "一个 AI 干活太慢了" | 装 **oh-my-openagent**（多 AI 并行） |
| "我想有自己的 skill" | 学 `/write-a-skill` 创建自己的 skill |
| "国内网络连 GitHub 太慢" | 配置国内镜像源 + 看本项目（汉化版） |

## 第四部分：避坑指南（必读）

### 安全类

1. **不要信任 AI 写的认证/支付/安全代码**
   - AI 有可能写出有漏洞的登录、支付逻辑
   - 这些代码**必须**一行一行人工 Review
2. **不要直接执行 AI 给你的 rm/delete 命令**
   - AI 说"删除这个文件"，你先看看这个文件是什么
   - 先让 AI 解释要删什么，你确认了再手动执行
3. **不要给 AI 你的 API Key 或密码**
   - 它不需要。配置写环境变量，别写在对话里

### 效率类

4. **对话 40-50 轮后 AI 开始"失忆"**
   - 解决方案：每 30 轮左右输一次 `/handoff`，开新对话接续
   - 别硬撑到 AI 东拉西扯才换
5. **AI 的知识有截止日期**
   - 比如它可能不知道某个库的最新版本 API
   - 如果你发现它用了不存在的函数，可能是版本过时了
   - 解决方案："查一下 xxx 的最新文档"
6. **不要把 AI 当搜索引擎**
   - AI 会"编造"看起来合理但实际不存在的东西（幻觉）
   - 关键信息（API 文档、版本号、配置项）去官方文档核对

### 学习类

7. **学新语言/框架的前 3 天不要用 AI**
   - 你亲手写 3 天代码，建立"手感"
   - 之后再让 AI 加速——但你得能看懂它写的
8. **看不懂 AI 的代码 = 你没有学到东西**
   - 每段看不懂的代码都让 AI 解释
   - 直到你理解了才继续
9. **不要同时装超过 3 个 skill 集合**
   - 冲突、占 context、互相覆盖
   - 少即是多

### 工具选择类

10. **不要同时装 superpowers 和 oh-my-openagent**
    - 两个都有自动触发机制，会打架
    - 二选一
11. **不要从非官方网站下载安装包**
    - 有人伪造 "oh-my-opencode 官方下载页" 钓鱼
    - 只从 GitHub 官方仓库或 npm 装
12. **Windows 用户注意**
    - OpenCode 在 Windows 上完全可用
    - 装的时候可能提示你装 OpenSSH——这是正常的，点确定

## 第五部分：常见问题速查

**Q：AI 写出来的代码跑不起来怎么办？**
A：把报错信息复制粘贴回去："这个报错是什么意思？怎么改？"

**Q：AI 突然用英文回答了怎么切回中文？**
A：说："请用中文回答"

**Q：我想让 AI 记住我的习惯**
A：在 CLAUDE.md 里加一行："注意：变量名用驼峰命名，函数名用动词开头"

**Q：对话太长 AI 变笨了怎么办？**
A：说：`/handoff` → 新对话 → "接续工作"

**Q：我该买哪个 AI 模型？**
A：新手先用 OpenCode 自带模型（免费）。觉得不够快再考虑 Claude Pro（$20/月）。别一上来就买最贵的。

**Q：CLAUDE.md 放在哪里？**
A：你的项目根目录，和 `package.json` / `README.md` 放在一起。

**Q：我该学哪个编程语言？**
A：AI 编程对语言不挑。但推荐 JavaScript/TypeScript 或 Python——AI 训练数据最多，写出来的质量最高。

## 总结一句话

> **第 1 周学怎么让 AI 干活，第 2 周学怎么让 AI 干好活，第 3 周了解还有啥好工具，第 4 周选你的升级路线。最重要的不是 AI 写得多快，而是你能不能看懂并控制它写的东西。**
